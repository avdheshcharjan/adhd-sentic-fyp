# Chapter 6: Conclusion and Future Work

This chapter summarises the contributions of the ADHD Second Brain project, evaluates the extent to which the original objectives were achieved, acknowledges the limitations of the current implementation, and outlines concrete directions for future research and development.

---

## 6.1 Summary of Achievements

This project set out to design and implement an on-device, AI-powered macOS application that would serve as a cognitive prosthetic for adults with Attention Deficit Hyperactivity Disorder (ADHD). The resulting system — ADHD Second Brain — demonstrated that it is feasible to deliver clinically-informed, privacy-preserving, and energy-efficient AI assistance on consumer hardware without reliance on cloud-based large language model inference. The application was built as a native macOS application that integrated an on-device large language model (Qwen3-4B, deployed via Apple's MLX framework \cite{hannun2023} \cite{yang2025}), a custom emotion classification pipeline, a long-term memory system, real-time activity monitoring, and a Just-In-Time Adaptive Intervention (JITAI) engine grounded in Barkley's five executive function domains \cite{barkley2010}. All five project objectives established at the outset of the project were met or exceeded, as summarised below.

The on-device language model achieved a throughput of 37.1 tokens per second on Apple Silicon hardware, with a peak resident set size of less than 3 GB and an energy consumption of 21,314 mJ per inference — corresponding to approximately 1.4% battery drain per hour of continuous use. These figures confirmed that quantised large language models, when deployed through hardware-optimised frameworks such as MLX, can deliver responsive conversational interactions without the latency, cost, or privacy implications of cloud inference \cite{lin2025}. The load-on-demand architecture ensured that the model occupied memory only during active interaction, preserving system resources for other tasks.

The emotion classification system represented a key technical contribution. A contrastive learning approach inspired by SetFit \cite{reimers2019} achieved 86% accuracy (0.862 macro-F1) using only 210 expert-labelled training sentences — a result that outperformed both a hybrid embedding-plus-SenticNet baseline (74%) and a fine-tuned DistilBERT classifier trained on 1,200 augmented samples (72%). The system classified user journal entries into six ADHD-relevant emotional categories (joyful, focused, frustrated, anxious, disengaged, and overwhelmed), enabling the JITAI engine to tailor interventions to the user's current affective state. The SenticNet 8 API \cite{cambria2024senticnet8} was orchestrated across 13 endpoints to provide fine-grained affective dimensions (pleasantness, attention, sensitivity, aptitude, and polarity) that enriched the context provided to the language model, achieving 100% API reliability across all benchmark runs.

The Mem0-based memory system \cite{mem0}, backed by pgvector for semantic retrieval, achieved 95% top-1 relevance across a curated evaluation set of 10 query–memory pairs, ensuring that the conversational agent could recall and reference prior interactions, user preferences, and emotional patterns over extended periods. The five-tier activity classification cascade processed application window titles at a throughput of 549 titles per second, with 78% of classifications resolved by deterministic rule-based matching before any machine learning inference was required — a design that minimised energy expenditure during continuous background monitoring. Concept Bottleneck Models provided explainable artificial intelligence (XAI) capabilities, enabling the system to surface human-interpretable intermediate concepts alongside its classification decisions. The entire system was validated through over 200 automated tests and a reproducible two-run benchmarking protocol that quantified cross-run variance for all latency-sensitive components.

## 6.2 Limitations

Despite the breadth of the system and the strength of its benchmark results, several limitations constrain the generalisability and clinical applicability of the current work. These limitations are acknowledged candidly, as they inform the directions for future research outlined in the subsequent section.

The most significant limitation was the absence of an Institutional Review Board (IRB)-approved user study with clinically diagnosed ADHD participants. All user interaction scenarios were evaluated through persona simulations rather than through longitudinal observation of real users in naturalistic settings. While the benchmark results demonstrated technical soundness — including classification accuracy, memory retrieval relevance, and inference throughput — they did not establish whether the system would meaningfully improve executive function, emotional regulation, or task completion rates for individuals with ADHD. Given that ADHD affects approximately 5% of adults worldwide \cite{faraone2021} and that digital interventions have shown preliminary evidence of efficacy \cite{garciaperal2025}, clinical validation remains an essential next step before any claims of therapeutic benefit can be made.

The emotion classifier, while achieving strong accuracy, was trained on expert-labelled data rather than clinically annotated corpora validated against standardised psychological instruments. The six-category emotion taxonomy was designed with ADHD-relevant affective states in mind, but it has not been validated against established clinical frameworks for emotional dysregulation in ADHD populations. Furthermore, the evaluation dataset comprised only 50 test sentences, which, while sufficient to demonstrate relative performance differences between approaches, limits the statistical power of the accuracy estimates. Similarly, the activity classification benchmark used 200 application titles, and the memory retrieval evaluation used 10 query–memory pairs — sample sizes that were adequate for development-stage validation but would need to be substantially expanded for publication-quality claims.

From a technical standpoint, several architectural constraints limited the system's reach and performance. The application was restricted to macOS, as it depended on Apple's Metal GPU backend via the MLX framework; no Windows, Linux, or mobile variant was developed, which limited accessibility for the broader ADHD population. The MLX Metal backend was not thread-safe, which prevented concurrent inference and required serialised access to the language model — a constraint that introduced queuing delays during periods of high interaction frequency. The SenticNet API, while achieving 100% reliability, was an external cloud dependency that accounted for 51.7% of the total pipeline latency; any disruption to this service would degrade the quality of affective context provided to the language model. Cross-run latency variance for cloud-dependent components (Mem0 and SenticNet) ranged from 8% to 40%, introducing unpredictability into the user experience that could not be fully mitigated at the application level. The resident set size measurements captured by psutil did not account for Metal GPU buffer memory, meaning that the reported peak RSS of less than 3 GB likely underestimated total memory consumption. The system was limited to English language processing, the Whoop biometric integration required a paid subscription, and all data remained siloed on a single device with no synchronisation across devices.

## 6.3 Future Work

The limitations identified in the preceding section point to several concrete and actionable directions for future research and development, spanning clinical validation, platform expansion, performance optimisation, and model improvement.

The highest-priority future work item is an IRB-approved user study with ADHD-diagnosed participants. A longitudinal study design with 20 to 30 participants, conducted over a period of 8 to 12 weeks, would enable assessment of the system's impact on executive function outcomes using validated instruments such as the Barkley Deficits in Executive Functioning Scale \cite{barkley2010}. Participants would use the application in their daily routines while quantitative metrics (task completion rates, journaling frequency, intervention acceptance rates) and qualitative feedback (semi-structured interviews) were collected. Such a study would establish whether the technical capabilities demonstrated in the benchmarks translate into meaningful improvements in the lived experience of individuals with ADHD, and would position the system within the growing body of evidence for digital ADHD interventions \cite{garciaperal2025}.

Cross-platform expansion represents a natural extension of the current work. An iOS companion application, sharing the same backend logic and memory system, would enable users to access their ADHD Second Brain across devices — a critical capability given that ADHD-related challenges do not confine themselves to desktop computing contexts. The shared backend architecture, with Mem0 \cite{mem0} and pgvector handling memory persistence, would facilitate synchronisation of conversational history and emotional patterns across platforms. A Linux port, while less immediately impactful for the target user population, would broaden the research community's ability to replicate and extend the work. Eventual App Store deployment and a public beta programme would provide the scale of user feedback necessary to refine the intervention strategies and classification models.

Several performance optimisations were identified through the benchmarking process that could substantially reduce perceived latency. The Mem0 memory store operation, which accounted for 38.8% of total pipeline latency, could be converted to an asynchronous fire-and-forget pattern, reducing perceived end-to-end latency by an estimated 39% without sacrificing memory persistence reliability. Local caching or knowledge distillation of SenticNet responses for commonly encountered emotional expressions would reduce the 51.7% latency contribution of the SenticNet API calls, while also eliminating the system's dependency on an external cloud service \cite{cambria2024senticnet8}. Parallel execution of SenticNet API calls and LLM inference — currently serialised due to the pipeline architecture — would overlap the two most latency-intensive operations and further reduce perceived response time. Together, these three optimisations could reduce the median end-to-end latency from its current level to a sub-second target that would feel instantaneous to users.

On the model improvement front, the contrastive emotion classifier's strong performance with only 210 training sentences suggests that further accuracy gains are achievable through careful data curation rather than bulk data generation. The project's experience with LLM-generated training data — which caused accuracy to regress from 86% to 82% due to dilution of the anxious class signal — demonstrated that data quality dominates data quantity for few-shot contrastive learning. Future work should focus on curating additional training sentences through structured elicitation from ADHD-diagnosed individuals, ensuring balanced representation across all six emotion categories and including boundary cases that the current classifier misclassifies. Fine-tuning a DistilBERT model on a carefully expanded and balanced dataset (rather than bulk-augmented data) remains a viable alternative path. Federated learning techniques could enable collaborative model improvement across multiple users' devices without centralising sensitive emotional data, preserving the privacy guarantees that motivated the on-device architecture. Multi-language emotion classification, beginning with major world languages, would extend the system's accessibility beyond English-speaking populations and could leverage multilingual sentence transformers \cite{reimers2019} as a starting point.

Finally, integration with Apple HealthKit would enable the system to incorporate physiological signals — heart rate variability, sleep duration, and activity levels — as additional inputs to the JITAI engine, potentially improving the timeliness and relevance of interventions by detecting physiological correlates of ADHD-related states before they manifest behaviourally. Clinical collaboration with ADHD researchers would ensure that such integrations are grounded in evidence-based practice and that the system's intervention strategies align with current therapeutic frameworks.

## 6.4 Lessons Learned

The development of ADHD Second Brain yielded several technical and methodological lessons that may be of value to researchers and practitioners undertaking similar projects at the intersection of on-device machine learning, affective computing, and assistive technology.

The most consequential lesson concerned the relationship between training data quality and model performance. The contrastive emotion classifier achieved its best accuracy (86% macro-F1) with 210 carefully curated, expert-labelled sentences. When 288 additional sentences generated by a large language model were added to the training set, accuracy regressed to 82%, with the anxious class F1 score dropping from 0.80 to 0.62. This regression was traced to class imbalance in the generated data (the frustrated and overwhelmed categories received 98 sentences each, while the anxious category received only 83) and to a dilution of the distinctive linguistic signal for anxiety-related expressions. The lesson was clear: for few-shot contrastive learning, a small number of high-quality, carefully balanced training examples outperformed a larger but noisily generated dataset. This finding aligns with broader observations in the few-shot learning literature \cite{reimers2019} and has direct implications for practitioners considering synthetic data augmentation.

On-device deployment constraints, far from being purely restrictive, drove creative engineering solutions. The load-on-demand architecture, 4-bit quantisation of the Qwen3-4B model \cite{yang2025}, and the five-tier activity classification cascade (which resolved 78% of classifications through deterministic rules before invoking any machine learning model) were all direct responses to the memory, energy, and latency budgets imposed by consumer hardware \cite{lin2025}. These constraints forced a discipline of efficiency that resulted in a system consuming only 21,314 mJ per inference — a figure that would have been far higher had the architecture been designed for server-class hardware and later ported to edge devices.

The modular architecture — in which the emotion classifier, memory system, activity monitor, SenticNet orchestrator, and language model were developed and benchmarked as independent components — proved essential for iterative development. Each component could be evaluated, optimised, and replaced without affecting others, and the two-run reproducibility protocol caught performance regressions early rather than allowing them to compound into late-stage surprises. A related lesson concerned the role of SenticNet features in the classification pipeline: while SenticNet's affective dimensions degraded classification accuracy when used as input features (reducing the hybrid classifier from 74% to 70%), they substantially enriched the context provided to the language model for generating empathetic and ADHD-aware responses \cite{cambria2024senticnet8}. This decoupling of affective analysis from classification — using different tools for different purposes within the same pipeline — was a key architectural insight.

Finally, the observation that embedding quality dominated classifier head configuration — all learning rate configurations produced identical 86% accuracy for the single-epoch contrastive model — reinforced the importance of selecting a strong pretrained sentence encoder \cite{reimers2019} and investing effort in the contrastive training procedure rather than in hyperparameter tuning of downstream classification heads. This finding has practical implications for resource-constrained research teams: time is better spent curating training pairs and selecting appropriate loss functions than conducting exhaustive grid searches over classifier hyperparameters.
